# CPU 스케쥴링 알고리즘

## FCFS (First-Come First-Served)

- 먼저 작업이 들어온 순서대로 처리
- EX: 선착순으로 상담을 받는 제도
- 단점: 먼저 들어온 작업이 매우 긴 작업이면, 상대적으로 빠르게 처리되는 작업들도 대기해야함

## SJF (Shortest Job First)

- 가장 짧은 작업부터 우선 처리
- EX: 가장 상담이 짧게 끝날 학생부터 먼저 진행
- 단점: 상대적으로 긴 작업은 계속 기다려야할수있음

## RR (Round Robin)

- 각 프로세스에 동일한 시간 부여
- EX: 똑같이 5분씩 상담 시간
- 단점: 시간 기준을 어떻게 잡느냐에따라 성능 이슈가 발생할수있음
