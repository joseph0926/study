# 실험 0.1 - 가설과 검증

## 실험 전 예측 (가설)

### 가설 1: 실행 순서
**예측**: 인라인 스크립트가 먼저 실행될 것
**근거**:
- HTML은 위→아래로 읽힘
- 인라인 스크립트가 더 위에 위치
- 외부 스크립트에 `defer`/`async` 없음 → 해당 라인에서 블로킹되어 파일을 가져올 것

**검증 포인트**:
- [ ] Console에서 실제 로그 순서 확인
- [ ] 인라인이 정말 먼저인가?

---

### 가설 2: HTML 파싱 중단 여부
**예측**: `<script src>` 만나면 파싱을 멈추고 JS 다운로드
**근거**:
- `defer`/`async` 속성이 없으면
- HTML 파싱과 JS 다운로드가 병렬로 진행되지 않음

**검증 포인트**:
- [ ] Network 탭의 타임라인 확인
- [ ] HTML 요청이 완료된 후 JS 요청이 시작되는가?
- [ ] JS 다운로드 중에 다른 리소스 요청이 있는가?

**🤔 추가 질문**:
- "파싱을 멈춘다"는 것은 정확히 무엇을 의미하는가?
- 브라우저가 HTML 읽기를 완전히 중단하는가?
- 아니면 렌더링만 중단되는가?

---

### 가설 3: Network 탭 표시
**예측**:
- ✅ `.html` 파일 요청 보임
- ✅ `.js` 파일 요청 보임
- ❌ 인라인 스크립트는 별도 요청 안 보임

**근거**:
- 인라인 스크립트는 이미 HTML에 포함되어 있음
- 별도의 네트워크 요청이 필요 없음

**검증 포인트**:
- [ ] Network 탭에서 실제 요청 개수 세기
- [ ] 각 요청의 크기, 타이밍 확인
